# 性能分析技能

## 核心指标

| 指标 | 健康范围 | 告警阈值 |
|------|---------|---------|
| 平均响应时间 | < 500ms | > 2000ms |
| P95 响应时间 | < 1000ms | > 3000ms |
| P99 响应时间 | < 2000ms | > 5000ms |
| 错误率 | < 0.1% | > 1% |
| 吞吐量 (QPS) | 业务峰值的 2 倍 | 低于峰值 |
| CPU 使用率 | < 60% | > 80% |
| 内存使用率 | < 70% | > 85% |

## 瓶颈检测流程

### 1. 响应时间分析
- 平均/P50/P95/P99 分布
- 慢请求 Top 10
- 时间段趋势 (是否有劣化)

### 2. 资源分析
- CPU: 是否持续高负载
- 内存: 是否有泄漏趋势 (持续增长不释放)
- 磁盘: I/O 等待时间
- 网络: 带宽利用率

### 3. 应用层分析
- 慢 SQL 查询 (> 1s)
- 线程池耗尽
- 连接池耗尽
- GC 停顿

## 优化建议模板

| 瓶颈 | 优化方案 | 预期效果 |
|------|---------|---------|
| 慢 SQL | 添加索引 + 优化查询 | 响应时间降低 50% |
| 高 CPU | 算法优化 + 缓存 | CPU 降低 30% |
| 内存泄漏 | 排查未关闭资源 | 内存稳定 |
| 高延迟 | CDN + 异步处理 | 用户感知延迟降低 |

}
