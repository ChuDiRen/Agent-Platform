# 测试用例生成技能

## 设计方法 (必须遵循)

### 1. 等价类划分
将输入分为有效等价类和无效等价类，每类取一个代表值:
- **有效**: 正常用户名、合法密码
- **无效**: 空值、超长、特殊字符、SQL注入

### 2. 边界值分析
对每个数值/长度约束，测试边界:
- 最小值 -1 / 最小值 / 最小值 +1
- 最大值 -1 / 最大值 / 最大值 +1
- 空、null、0

### 3. 错误推测
根据经验推测易错场景:
- 并发操作、网络中断、会话过期
- 浏览器回退、多标签页、刷新页面
- 权限变更、数据被他人修改

## 用例模板

```json
{
  "id": "TC-001",
  "name": "验证使用正确账号密码可成功登录",
  "priority": 1,
  "precondition": "系统中存在已注册账号 test@example.com",
  "steps": "1. 打开登录页\n2. 输入用户名 test@example.com\n3. 输入密码 Test@1234\n4. 点击登录",
  "expected": "跳转到首页，显示用户名",
  "type": "positive"
}
```

## 优先级定义

| 级别 | 含义 | 覆盖要求 |
|------|------|---------|
| P0 | 核心业务流程 | 必须全部通过 |
| P1 | 分支覆盖 | 主要分支必须覆盖 |
| P2 | 边界与异常 | 关键边界必须覆盖 |
| P3 | 兼容性与体验 | 尽量覆盖 |

## 覆盖率检查
- 每个需求点至少 1 个正向 + 1 个反向用例
- 每个输入字段至少 1 个边界用例
- 每个接口至少 1 个鉴权 + 1 个参数校验用例
